### ⏱️ 4:35–5:00　ContentProvider 功能

🖥️ **画面**：可切到命令行终端跑 adb，或停留在看板页口头说明

🎙️ **口播**：
> 关于 ContentProvider，应用对外暴露了日程数据，URI 是 content://com.qiu.aischedule.provider/events，实现了增、删、改、查全部四项操作。刚才看板页的列表数据，就是通过 ContentProvider 读取的。
>
> （可选，切到命令行）我也可以用 adb 命令直接从外部查询验证：`adb shell content query --uri content://com.qiu.aischedule.provider/events`，回车，可以看到数据库里的所有日程数据。

#### 📋 这一步具体怎么做（录前务必自己先跑通一遍）

**最稳的演示闭环（约 25 秒）**：App 里先有一条日程 → 切到 AS 底部 Terminal（或 PowerShell）→ 敲下面的 query → 镜头对准输出，能看到刚存那条日程的 title/startTime/location → 证明「App 写入的数据，通过 ContentProvider 对外可读」。

```bash
adb devices                                                # 0. 确认设备连着
adb shell content query --uri content://com.qiu.aischedule.provider/events        # 1. 查全部（最稳，只演示这条）
adb shell content query --uri content://com.qiu.aischedule.provider/events/1      # 2. 查单条
# 3.（可选）经 CP 插入——字段名要与代码一致，标题用英文避免中文乱码
adb shell content insert --uri content://com.qiu.aischedule.provider/events \
  --bind title:s:adbTest --bind startTime:l:1750000000000 \
  --bind endTime:l:1750003600000 --bind location:s:Lab \
  --bind reminderMinutes:i:15 --bind sourceText:s:adb --bind status:s:saved
# 4.（可选）经 CP 删单条
adb shell content delete --uri content://com.qiu.aischedule.provider/events/1
```

> insert/delete 后看板页会因 `notifyChange` 自动刷新——正好演示「外部经 CP 改数据 → App 内自动刷新」，加分。

**前提与坑**：
- **adb 不在 PATH**：用完整路径 `%LOCALAPPDATA%\Android\Sdk\platform-tools\adb.exe` 替换命令里的 `adb`。
- **中文标题易翻车**：Windows 下 `adb shell` 传中文常乱码 → insert 标题用英文，或只演示 query。
- **多设备**：同时连模拟器+真机时，命令前加 `-s emulator-5554` 指定。
- **query 为空**：先在 App 存至少一条日程再查。
- **不想敲命令**：原件允许「展示**或**说明」，纯口头讲「看板页就是 `contentResolver.query` 经此 URI 读取，代码在 `ScheduleListActivity.java:124`」也合规。

**呈现技巧**：终端字体调大，敲完命令停 2 秒让评审看清输出。
