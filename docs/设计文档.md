## 1. 项目名称、主题、面向用户与使用场景

- **项目名称**：AI日程转换助手（AISchedule）
- **主题**：智能日程管理 App——把一句口语化表达，经 LLM 解析为结构化日程草稿，用户确认后写入本地日历并到点提醒。
- **面向用户**：大学生（课程/作业/社团/小组会议）、上班族（会议/截止日期）、健忘型用户。
- **使用场景**：聊天或临时想到一件事，直接说一句话即可生成可编辑、可提醒的日程，免去手动逐项填写。

## 2. 主要功能模块（6 个核心功能）

1. 一句话创建日程（AI 提取标题/时间/地点/提醒）
2. AI 澄清与纠错（时间含糊时追问）
3. 日历/列表双视图（CalendarView + RecyclerView）
4. 自然语言修改
5. 提醒通知（Notification）
6. API Key 与隐私设置（本地加密保存）

## 3. 主要页面设计说明（≥3 个）

- **首页输入页** `MainActivity`：一句话输入 + "AI 解析"按钮。
- **AI 确认页** `AiConfirmActivity`：展示并可编辑解析字段，确认保存（含手动填写降级）。
- **日历看板页** `ScheduleListActivity`：CalendarView + 当天日程 RecyclerView。
- **详情编辑页** `DetailActivity`：查看/修改/删除。
- **解析历史页** `HistoryActivity`：原句 + JSON + 可信度（RecyclerView）。
- **设置页** `SettingsActivity`：provider/baseUrl/model/Key 配置与测试。

## 4. 数据库设计（Room 三表）

- **EventRecord（日程表）**：id、title、startTime、endTime、location、reminderMinutes、sourceText、status。
- **ParseHistory（解析历史）**：id、inputText、modelName、jsonResult、confidence、createdAt、isApplied。
- **ApiConfig（配置）**：id、provider、baseUrl、modelName、keyAlias、createdAt、updatedAt。
- DAO 提供 `@Insert/@Update/@Delete/@Query`，读方法返回 `LiveData<List<EventRecord>>`；`AppDatabase` 为单例。
- （具体字段类型与 DAO 方法签名见 `data/local/` 源码，阶段 1 完成后补充。）

## 5. 关键技术实现

- **RecyclerView**：日程列表（标题/时间/地点/提醒）与解析历史列表（原句/JSON/可信度）。
- **Room + CRUD**：增查改删全实现；写入后台线程，读取用 LiveData 自动刷新。
- **ContentProvider**：`ScheduleProvider` 经 URI 暴露 events 表，实现 query/insert/update/delete；看板页经 ContentResolver 读取。
- **网络访问**：OkHttp `enqueue` 异步调用 DeepSeek `chat/completions`，Gson 解析回填确认页；不在主线程。
- **Notification**：AlarmManager 定时 + BroadcastReceiver 发通知，点击跳详情；另设"立即测试提醒"。

## 6. 项目中遇到的问题与解决方式（≥3 个）

1. （阶段 1）主线程访问 Room 报 `IllegalStateException` → 改为后台线程（ExecutorService）执行写操作。
2. （阶段 5）API 26+ 通知不显示 → 创建 NotificationChannel；并对 minSdk 24 用 `SDK_INT >= O` 判断。
3. （阶段 4）LLM 偶尔返回非 JSON 或带说明文字 → 提取 `{...}` 段落再 Gson 解析，失败降级手动填写。
